# Situação de Aprendizagem – Git e GitHub

## Integrantes

- Jéssica Santos de Brito
- Joel Carlos Barbosa e Lima 

## Objetivo

O objetivo desta atividade foi colocar em prática os conhecimentos adquiridos sobre Git e GitHub, 
aprendendo a utilizar o controle de versão para desenvolver um projeto em equipe.
Durante a atividade, trabalhamos com repositórios, commits, branches, sincronização entre repositório 
local e remoto e integração das alterações realizadas por cada integrante.

## Desenvolvimento

Inicialmente foi  criado um repositório no GitHub para armazenar o projeto.
Em seguida, o repositório foi clonado para o computador de desenvolvimento, 
permitindo que as alterações fossem realizadas localmente.

Ao longo da atividade, cada integrante realizou modificações no
projeto e registrou essas alterações por meio de commits, mantendo um histórico das mudanças realizadas.
Também utilizamos branches para organizar o trabalho em equipe, 
permitindo que cada integrante desenvolvesse sua parte separadamente antes da integração ao projeto principal.

Após a conclusão das alterações, elas foram incorporadas ao repositório utilizando os comandos do Git e enviadas ao GitHub.
Essa prática permitiu compreender, na prática, como funciona o versionamento de código e a colaboração entre diferentes
desenvolvedores em um mesmo projeto.

## Dificuldades Encontradas
Durante o desenvolvimento do projeto enfrentamos algumas dificuldades relacionadas ao 
uso das branches e à organização do fluxo de trabalho.

Em um dos momentos da atividade, uma alteração foi realizada diretamente na branch main.
Com a orientação do professor, entendemos por que essa não era a melhor prática e
aprendemos como organizar corretamente o desenvolvimento utilizando branches para cada 
integrante.

Também tivemos alguns desafios durante a sincronização entre o repositório local e o
remoto, principalmente ao enviar e atualizar as alterações. Essas situações foram importantes para compreendermos melhor o 
funcionamento do Git e reforçaram a importância de seguir um fluxo de trabalho organizado para evitar conflitos.
Esses desafios fizeram parte do aprendizado e contribuíram para que concluíssemos a 
atividade com uma compreensão muito maior sobre o uso do Git e do GitHub.

## Aprendizados

Ao finalizar esta atividade, foi possível compreender melhor a importância do controle de versão no 
desenvolvimento de software. Além de aprender os principais comandos do Git, 
também entendemos como o GitHub facilita o trabalho em equipe, permitindo que várias pessoas contribuam
para o mesmo projeto de forma organizada.

A atividade também mostrou que cometer erros faz parte do processo de aprendizagem.
Resolver esses problemas com a orientação do professor foi essencial para consolidar o 
conhecimento adquirido durante a disciplina.

## Conclusão

A Situação de Aprendizagem foi fundamental para transformar os conteúdos estudados em prática.
O projeto permitiu vivenciar situações reais de desenvolvimento colaborativo, desde a criação do 
repositório até a integração das alterações realizadas pelos integrantes.
Ao final da atividade, conseguimos compreender melhor o funcionamento do Git e do GitHub,
reforçando a importância do controle de versão para a organização, segurança e evolução de 
projetos de software.
